## What is KemyCard?

KemyCard is an API that allows businesses to issue **virtual and physical cards**, manage **KYC/POA verification**, create **crypto wallets**, **virtual bank accounts**, and purchase **gift cards** — all through a simple, unified REST API.

## Base URL

```
https://api.kemycard.com/api/v1
```

## Authentication

All requests require an `Authorization` header with your API key:

```bash
Authorization: Bearer sk_test_YOUR_API_KEY
```

<Info>
  You have two types of keys:
  - `sk_test_...` for test mode (simulated data, zero impact)
  - `sk_live_...` for production (real operations)
</Info>

## Who is this API for?

- **Fintechs** looking to offer cards to their users
- **Neobanks** that need a card-issuing infrastructure
- **Crypto platforms** that want cards funded with USDC
- **Marketplaces** looking to distribute gift cards or prepaid cards
