Let's create a customer using your test key. No real data will be stored — everything is simulated in test mode.

## 1. Create a customer

<CodeGroup>

```bash cURL
curl -X POST https://api.kemycard.com/api/v1/customers \
  -H "Authorization: Bearer sk_test_YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "external_reference": "user-001",
    "first_name": "John",
    "last_name": "Doe",
    "email": "john@example.com",
    "phone": "+33612345678",
    "date_of_birth": "1990-05-15",
    "address": "123 Main Street",
    "city": "Paris",
    "state": "IDF",
    "postal_code": "75001",
    "country": "FRA"
  }'
```

```php PHP
$ch = curl_init('https://api.kemycard.com/api/v1/customers');
curl_setopt_array($ch, [
    CURLOPT_POST => true,
    CURLOPT_HTTPHEADER => [
        'Authorization: Bearer sk_test_YOUR_API_KEY',
        'Content-Type: application/json',
    ],
    CURLOPT_POSTFIELDS => json_encode([
        'external_reference' => 'user-001',
        'first_name' => 'John',
        'last_name' => 'Doe',
        'email' => 'john@example.com',
        'phone' => '+33612345678',
        'date_of_birth' => '1990-05-15',
        'address' => '123 Main Street',
        'city' => 'Paris',
        'state' => 'IDF',
        'postal_code' => '75001',
        'country' => 'FRA',
    ]),
    CURLOPT_RETURNTRANSFER => true,
]);
$response = json_decode(curl_exec($ch), true);
```


```python Python
data = response.json()
```

## 3. What's next?

Now that you have a customer, here's the typical flow:

<Steps>
  <Step title="Verify identity (KYC)">
    Submit a KYC verification to upgrade the customer from `basic` to `verified`.
  </Step>
  <Step title="Issue a card">
    Create a virtual Mastercard or Visa card for the verified customer.
  </Step>
  <Step title="Fund the card">
    Top up the card from your ops balance.
  </Step>
</Steps>
